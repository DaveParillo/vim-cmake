INTRODUCTION                                    *cmake*

When working on a CMake project, this plugin provides commands to configure
and build CMake projects without leaving Vim. The plugin automatically locates
the build directory (see |g:cmake_build_dir|), persists that location across
sessions, and wires up |'makeprg'| so that |:make| builds in the correct
directory.

==============================================================================
COMMANDS                                        *cmake-commands* *:CMakeBuild*

:CMakeBuild [target] [args]
                        CMakeBuild searches for the configured build directory
                        and if found updates the |:makeprg| option and then
                        builds the [target] using |:make|.

                        All arguments after the first are directly passed on
                        to CMake.

                        The first parameter must be a target name and other
                        cmake parameters can follow the target in any order.

                        Use |:CMakeConfigure| to find a build directory,
                        generate toolchain files, and update the
                        *cmake-cache* file.

                        Tab completion is available for [target]: >
                            :CMake my_<Tab>
<
                        When invoked without arguments, the default build
                        target is used.

                                                *:CMakeConfigure*
:CMakeConfigure [args]  Run cmake configuration in the build directory,
                        passing [args] directly to cmake.
                        CMakeConfigure searches for the build directory
                        before writing configuration files there

                        Build directory discovery follows this order:
                          1. Use |b:build_dir| if already set.
                          2. Restore from the *cmake-cache* (.vim-cmake).
                          3. Search upward from the current file for a
                             directory named |g:cmake_build_dir|.

                        Once found, the build directory is saved to the
                        *cmake-cache* automatically.

                        Use this to
                        explicitly re-run configuration, for example after
                        changing a compiler or build type: >
                            :CMakeConfigure -DCMAKE_BUILD_TYPE=Debug
<
                        Plugin global variables are translated into
                        cmake arguments automatically when set. See
                        |cmake-variables| for the full list.

                                                *:CMakeClean*
:CMakeClean!            Delete all files in the build directory, equivalent
                        to a cmake clean. Prompts for confirmation before
                        proceeding.

                        Add [!] to skip the confirmation prompt: >
                            :CMakeClean!
<
                        The command refuses to delete the build directory if:
                          - It does not contain a CMakeCache.txt (unless [!]
                            is given).
                          - It resolves to the home directory, current working
                            directory, or filesystem root.

                        After a successful clean, |b:build_dir| is unset and
                        the session cache is cleared, so the next |:CMake|
                        call will re-run discovery and configuration.

                                                *:CMakeFindBuildDir*
:CMakeFindBuildDir      Clear the current build directory path for the current
                        buffer and the session cache, then search for a new one.
                        Useful after creating a new build directory or when
                        the wrong directory was previously detected.

==============================================================================
OPTIONS                                         *cmake-options*

                                                *g:cmake_build_dir*
g:cmake_build_dir       The directory name to search for when locating the
                        build directory. Searched upward from the current
                        file. Default: 'build'. >
                            let g:cmake_build_dir = 'build'
<
                                                *b:build_dir*
b:build_dir             The resolved path to the cmake build directory for
                        the current buffer. Set automatically by |:CMake| or
                        |:CMakeFindBuildDir|. Can be set manually to override
                        automatic discovery: >
                            let b:build_dir = '/path/to/my/build'
<
                        Once set it is not overwritten unless you call
                        |:CMakeFindBuildDir| or set it explicitly.

                                                *g:cmake_install_prefix*
g:cmake_install_prefix  Passed to cmake as -DCMAKE_INSTALL_PREFIX.

                                                *g:cmake_build_type*
g:cmake_build_type      Passed to cmake as -DCMAKE_BUILD_TYPE. >
                            let g:cmake_build_type = 'Debug'
<
                                                *g:cmake_cxx_compiler*
g:cmake_cxx_compiler    Passed to cmake as -DCMAKE_CXX_COMPILER. Changes
                        take effect after |:CMakeClean| and |:CMakeConfigure|.

                                                *g:cmake_c_compiler*
g:cmake_c_compiler      Passed to cmake as -DCMAKE_C_COMPILER. Changes
                        take effect after |:CMakeClean| and |:CMakeConfigure|.

                                                *g:cmake_build_shared_libs*
g:cmake_build_shared_libs
                        Passed to cmake as -DBUILD_SHARED_LIBS.

                                                *g:cmake_toolchain_file*
g:cmake_toolchain_file  Passed to cmake as -DCMAKE_TOOLCHAIN_FILE.

                                                *g:cmake_project_generator*
g:cmake_project_generator
                        Passed to cmake as -G. Changes take effect after
                        |:CMakeClean| and |:CMakeConfigure|.

                                                *g:cmake_export_compile_commands*
g:cmake_export_compile_commands
                        Passed to cmake as -DCMAKE_EXPORT_COMPILE_COMMANDS.
                        Default: 0.

                                                *g:cmake_ycm_symlinks*
g:cmake_ycm_symlinks    When set to 1, creates a symlink to the generated
                        compile_commands.json in the project root for use
                        with YouCompleteMe or any LSP client that expects it
                        there. Default: 0.

                                                *g:cmake_use_smp*
g:cmake_use_smp         When set to 1, passes -j{nproc} to the build command
                        to enable parallel builds. Requires `nproc` to be
                        available on PATH. Default: 0.

                                                *g:cmake_usr_args*
g:cmake_usr_args        Arbitrary extra arguments passed to cmake during
                        configuration. Appended after all other arguments: >
                            let g:cmake_usr_args = '-DSOME_OPTION=ON'
<

==============================================================================
SESSION CACHE                                           *cmake-cache*

The plugin persists |b:build_dir| across sessions using a dotfile named
.vim-cmake written to the project root (one level above the build directory).
This file is read automatically on |BufReadPost| and written on |BufWritePost|.

The cache is cleared automatically by |:CMakeFindBuildDir| and |:CMakeClean|.
It is also invalidated silently if the recorded directory no longer exists.

The .vim-cmake file is a per-developer local setting and should be added to
the project .gitignore: >
    echo '.vim-cmake' >> .gitignore
<
